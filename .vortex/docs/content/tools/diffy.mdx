# Diffy - Visual regression testing

**Vortex** integrates with [Diffy](https://diffy.website) for visual regression
testing. After each deployment, a comparison can be triggered automatically (or
manually) between the deployed environment and a baseline, with the result
posted back to the related pull request.

For the end-to-end workflow, triggers, and decision points, see
[Visual regression](../development/visual-regression.mdx) under Development.

## Account setup

1. Create a Diffy account at [app.diffy.website](https://app.diffy.website).
2. Create a project for your site.
3. Configure the `production` (and optionally `staging`, `development`)
   environment URLs inside the Diffy project settings.
4. Generate an API key from
   [app.diffy.website/#/keys](https://app.diffy.website/#/keys).

## Repository configuration

The shipped `test-vr.yml` workflow reads configuration from GitHub Actions
secrets and repository variables.

### Secrets

| Name | Required | Purpose |
|---|---|---|
| `VORTEX_VR_DIFFY_API_KEY` | yes | Diffy API key |
| `VORTEX_VR_DIFFY_PROJECT_ID` | yes | Numeric Diffy project ID (visible in the Diffy project URL) |

Add these under *Settings > Secrets and variables > Actions > Repository
secrets*.

### Variables

| Name | Default | Purpose |
|---|---|---|
| `VORTEX_VR_DIFFY_CLI_VERSION` | `0.1.53` | Pinned `diffy-cli` release used by both jobs |
| `VORTEX_VR_DIFFY_MAX_WAIT` | `2700` | Maximum seconds to wait for a comparison to complete (45 minutes) |
| `VORTEX_VR_DIFFY_POLL_INTERVAL` | `30` | Polling interval in seconds |
| `VORTEX_VR_DIFFY_PR_LABEL` | `VR` | PR label that opts a deployment into visual regression. Case-insensitive |
| `VORTEX_VR_DIFFY_AUTO_BRANCHES` | `deps/*` | Comma-separated glob list of PR head branches that bypass the label gate (matches Renovate's `branchPrefix`). Set to empty to require the label on every PR. |
| `VORTEX_VR_DIFFY_SOURCE_ENV` | `production` | Default Diffy source environment for comparisons |

Add these under *Settings > Secrets and variables > Actions > Repository
variables*.

### Deployment-side configuration

For automatic dispatch, the deployment environment (Lagoon post-rollout, Acquia
post-code-deploy hook, or similar) needs a GitHub token with `repo` scope
exposed as `VORTEX_NOTIFY_DIFFY_TOKEN` (falls back to
`VORTEX_NOTIFY_GITHUB_TOKEN` if set, then to `GITHUB_TOKEN`). The repository
name is taken from `VORTEX_NOTIFY_DIFFY_REPOSITORY` (falls back to
`VORTEX_NOTIFY_GITHUB_REPOSITORY`).

Add `diffy` to `VORTEX_NOTIFY_CHANNELS` to enable the dispatch:

```ini
VORTEX_NOTIFY_CHANNELS=email,github,diffy
```

## How to trigger a comparison

Two entry points are supported:

1. **Automatic**: open a PR, apply the `VR` label, push. The deployment
   completes, `notify-diffy` dispatches the workflow. The workflow then
   resolves the PR from the deployed commit SHA via the GitHub API, checks the
   `VR` label, runs the comparison, and posts a summary PR comment. Works
   identically across hosting providers - every host that exposes the deployed
   environment URL through `VORTEX_NOTIFY_ENVIRONMENT_URL` is supported.
2. **Manual**: run *Test - Visual regression* from the GitHub Actions UI with
   a target URL and an optional source environment. No PR is involved; the
   result lives in the Diffy UI.

To limit which branches dispatch the workflow at all (save GitHub Actions
minutes on hosts that deploy frequently), set
`VORTEX_NOTIFY_DIFFY_BRANCHES` to a comma-separated branch allowlist. Leave
empty to dispatch on every deployment - the workflow itself gates by PR +
label.

## Costs and quotas

Diffy charges per screenshot set. Vortex's default gating (`VR` label on PRs
only) keeps the run rate low. Increase or decrease the gate as needed for your
team's review rhythm.

## Disabling Diffy

Remove `diffy` from `VORTEX_NOTIFY_CHANNELS` and delete the
`.github/workflows/test-vr.yml` workflow file (or set
`VORTEX_VR_DIFFY_API_KEY` to empty - the workflow will fail noisily but no
Diffy API calls will be made).

To remove the integration entirely, re-run the installer and answer "no" to
the visual regression prompt.
