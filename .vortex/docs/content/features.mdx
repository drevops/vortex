    Modern Drupal 11 with pre-configured settings, module and theme scaffolds, and
    admin modules for production-ready development.

    ### Modern Composer-based architecture

    - Latest [Drupal 11](https://www.drupal.org/project/drupal) with security updates
    - Configurable webroot directory to fit specific hosting provider requirements
    - Modern PHP 8+ compatibility with type declarations

    ### Settings and environment detection

    - Pre-configured [general settings](https://github.com/drevops/vortex/blob/develop/web/sites/default/settings.php)
    - Environment type detection based on the hosting provider
    - Optimised per-module settings for quick start
    - Multi-database configuration for MySQL and MariaDB

    ### Module & theme scaffolds

    - [Custom module scaffold](https://github.com/drevops/vortex/blob/develop/web/modules/custom/ys_base)
    - [Custom theme scaffold](https://github.com/drevops/vortex/blob/develop/web/themes/custom/your_site_theme)
    - [Tests scaffold](https://github.com/drevops/vortex/blob/develop/web/modules/custom/ys_base/tests)

    ### Admin modules

    - [Environment indicator](https://www.drupal.org/project/environment_indicator) for visual identification
    - [Shield](https://www.drupal.org/project/shield) for HTTP authentication
    - [Stage file proxy](https://www.drupal.org/project/stage_file_proxy) for development efficiency
    - Configuration for [Search API](https://www.drupal.org/project/search_api) ([Solr](https://www.drupal.org/project/search_api_solr))
    - Configuration for caching ([Redis](https://www.drupal.org/project/redis))
    - Configuration for antivirus ([ClamAV](https://www.drupal.org/project/clamav))

  <VerticalTab>üèóÔ∏è Continuous Integration | Continuous integration configurations</VerticalTab>
  <VerticalTabPanel>

    Continuous integration and deployment with multiple provider support.

    ### Multiple CI providers

    - Identical CI configurations for continuous integration providers
    - [CircleCI](https://circleci.com/) pipeline configuration
    - [GitHub Actions](https://github.com/features/actions) pipeline configuration

    ### 3-stage pipeline

    - _Download the database_ from hosting and cache it for 24 hours for faster builds
    - _Build_ and test a fully provisioned site
    - _Deploy_ the site to the hosting provider

    ### Containerized environments

    - Drupal runs in a containerized stack
    - Identical stack is used for local development ensuring consistency
    - Identical PHP version, dependency versions, tooling and configurations as used during local development

    ### Pipeline flow control

    - Pipeline triggered by code pushes to pre-configured branches/tags or manually
    - Skip deployments for specific branches or pull requests
    - Selectively ignore code checks and test failures without modifying the pipeline configuration

    ### Assets and code coverage

    - Build and test assets are stored as pipeline artifact and accessible after the build
    - Code coverage reports are stored as pipeline artifact and accessible after the build
    - Code coverage reports are pushed to [Codecov](https://codecov.io/) for easy access and visualization.

    :::note

    [Azure Pipelines](https://azure.microsoft.com/) and [GitLab CI](https://about.gitlab.com/solutions/continuous-integration/) are coming soon

    :::
  </VerticalTabPanel>

    ### Identical workflows across environments

    - Scripted workflows for common tasks configured to run in the same way in all environments
    - Support for workflow customization via hooks
    - Database sourcing from hosting provider or intermediate storage (URL, FTP, container image, etc.)

    ### Deployment Notifications

    - Deployment notifications via email, GitHub, Jira, New Relic
    - Integration with GitHub to provide a deployed environment status
    - Integration with Jira to post deployed environment information as a comment
    - Integration with New Relic to mark deployments

    ### Automated dependency management

    - Automated dependency updates via [Renovate](https://www.mend.io/renovate/)
    - Separate schedules for critical and regular updates
    - Automated pull requests for dependency updates
    - Optional continuous integration job for self-hosted runner for projects restricted in terms of third-party access.

    ### Development workflow automation

    - [Pull request template](https://github.com/drevops/vortex/blob/develop/.github/PULL_REQUEST_TEMPLATE.md) for consistent reviews
    - [Release drafter](https://github.com/release-drafter/release-drafter) for automated changelog
    - [PR auto-assign](https://github.com/toshimaru/auto-author-assign) for review assignments
    - [PR auto-label](https://github.com/eps1lon/actions-label-merge-conflict) for merge conflicts

  <VerticalTab>üåÄ Vortex | Consistency across your projects</VerticalTab>
  <VerticalTabPanel>

    Vortex provides several features to ensure consistency and reliability
    across your Vortex-onboarded projects so that you can rely on future updates
    and improvements.

    ### Product maturity

    - **Vortex** is a mature product with over 7 years of development and
      production use across education, government, commercial, healthcare,
      not-for-profit, financial services, and technology sectors.
    - Constantly evolving with the latest Drupal, PHP, containerization, CI/CD
      practices, and tooling.
    - Managed as a product with a dedicated team and a [roadmap](/contributing/roadmap).
    - Released monthly with new features, improvements, and bug fixes.

    ### Multi-layered automated testing for every change

    - Test coverage for containerized stack for guaranteed readiness
    - Test coverage for tooling configuration for guaranteed compatibility
    - Unit tests for workflow scripts and coverage reporting for reliable and consistent execution
    - Functional test coverage for workflows: run all workflow commands as a developer would
    - Automated documentation site generation and validation for up-to-date and accurate documentation

    ### Vortex installation and updates

    - Standalone `installer` application for installing and updating Vortex projects
    - Install and/or update from the most recent version or a specific Vortex Git SHA
    - Comprehensive `installer` test suite to guarantee reliable project updates


    :::note

    Advanced web installer with automated end-to-end setup and project dashboard
    are coming soon
