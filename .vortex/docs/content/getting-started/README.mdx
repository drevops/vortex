# Overview

**Vortex** is a Drupal project template designed to simplify onboarding and
streamline long-term project maintenance. This page provides a high-level view
of the system — what it is, why it exists, and how its components work together.

## Why **Vortex** exists

Setting up Drupal projects can be time-consuming and inconsistent:

<ul style={{ listStyle: 'none', paddingLeft: 0 }}>
  <li>⏱️ Developers lose hours or days setting up their local, CI, and hosting environments</li>
  <li>⚠️ Inconsistent setups cause false negatives, undermining the reliability of tools</li>
  <li>🔌 Lack of understanding of third-party service inner workings slows integration and troubleshooting</li>
  <li>📉 Poor onboarding documentation affects team productivity</li>
</ul>

This leads to:

<ul style={{ listStyle: 'none', paddingLeft: 0 }}>
  <li>😩 Developer frustration and burnout</li>
  <li>⏳ Project delays and missed deadlines</li>
  <li>💸 Increased unplanned costs and budget overruns</li>
</ul>

**Vortex** was created to solve these challenges by providing a complete
foundation for developing and maintaining Drupal projects.

### 📦 Template

<ul style={{ listStyle: 'none', paddingLeft: 0 }}>
  <li>💧 A pre-configured Drupal project structure with module, theme, and test boilerplate</li>
  <li>🧰 Pre-configured tools for code quality checks and automated testing</li>
  <li>🐳 Docker-based environment for local development and cloud hosting</li>
  <li>☁️ Built-in support for Acquia and Lagoon cloud hosting platforms</li>
  <li>🏗️ CI/CD using GitHub Actions or CircleCI</li>
  <li>📖 Project documentation</li>
</ul>

### 📖 Documentation

<ul style={{ listStyle: 'none', paddingLeft: 0 }}>
  <li>🌐 Centralised documentation at https://vortex.drevops.com</li>
  <li>🗂️ Scaffolded structure for project-specific docs</li>
  <li>📋 Onboarding checklist to guide team setup</li>
</ul>

### 🎛️ Installer

<ul style={{ listStyle: 'none', paddingLeft: 0 }}>
  <li>🛠️ Standalone Symfony Console application</li>
  <li>🏗️ Applies structure and tokens during installation</li>
  <li>📦 Adds boilerplate files and custom selections</li>
  <li>🔌 Extensible with a flexible API for new integrations</li>
</ul>
